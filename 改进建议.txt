# 单兵作战

## 目标威胁评判系统

对于每个bot，引入目标威胁评判系统，每个AI内置一个可量化的敌对目标威胁指数。

如果可攻击范围内有敌对目标，并且随身携带的武器可以有效攻击这个敌对目标，那么攻击顺序如下：
1. 攻击对整个集体威胁更大的敌对目标，如坦克、直升机，或者一整队偷袭的敌对士兵。
2. 攻击那些攻击自己的敌对目标，不管这个敌对目标是否打中了自己，只要它朝自己方向攻击就算攻击自己的敌对目标
3. 攻击距离自己最近的敌对目标

## 手持武器判定系统

对于手持不同主武器的bot，要有不同的策略选择。

1.手持AK、喷子等近战武器的bot，尽可能在交战最前线，同时不放弃远程点射。
2.手持狙击枪（最大的那一把）的bot，尽可能的在后方进行点射，自行寻找掩体，同时注意不要脱离战场太远。
3.手持其余武器的bot，按照原来的模式攻击。

对于手持不同副武器的bot，也要有不同的策略。

1.手持火箭筒的bot，尽可能的去攻击坦克、直升机、四人车、二人车，大规模 敌方远距离人群，不远的后方支援前线冲锋。
2.手持手雷、闪光弹的bot，尽可能的去近战攻击敌对人群，或者在进攻前先轰炸对手。

## 单兵战术能力提升

1. 听声辩位，对枪声来源保持警戒。但不要过度警戒，破坏优先级更高的动作。
2. 换弹要退回掩体。要善于利用掩体，也要敢于冲锋，冲锋要善于利用地形。
3. 精度 0.8x, 反应时间 1.0x
4. 行进的时候注意警戒，善于利用掩体行军。

# 载具作战

有关载具，让bot尽可能的不要让载具空出来，如果发现己方重生点有能够使用的空缺载具，那么优先重生到这些 敌方并且一定要开载具。同时要注意人数的分配，对于远离战场的载具重生点，适当分配1-2个bot去即可。

## 坦克作战要求

坦克是最游戏中最为重型的载具。坦克炮手攻击目标的优先级如下：

1. 攻击对整个集体威胁更大的敌对目标，如坦克、直升机。
2. 四人车、二人车（如果在没有机枪手的情况下，优先攻击对面拿火箭筒瞄准自己的敌人）
3. 大群的敌对士兵
4. 其他目标（如敌方落单士兵）

坦克机枪手攻击目标的优先级如下：
1. 对坦克造成威胁的敌对手持火箭筒士兵
2. 大群的敌对士兵
3. 四人车、二人车
4. 敌对落单士兵
5. 敌方坦克直升机
6. 其他目标

坦克优先开往正面战场，正面冲锋。

## 直升机作战要求

直升机作为游戏中机动性最强的载具，直升机优先攻击目标如下：

1. 手持火箭筒的敌人
2. 敌方大群的士兵
3. 坦克、四人车、二人车
4. 敌方落单的士兵
5. 敌方直升机

直升机应该保持机动性和战斗参与度，时刻保持对地火力输出和对己方的火力支援。

## 四轮车、二轮车的作战要求

1. 要让AI能够在车上就看到前方障碍物学会躲避，而不是撞到障碍物上动不了。
2. 让一部分bot用适当的概率加快车速，提升效率。
3. 副座位士兵学会优先攻击敌方手持火箭筒的敌军士兵或者坦克。

# 战斗战术

## 步坦协同、步炮协同、步机协同

步坦协同是比较重要的改进，在游戏中，可以简化为：
1. 己方步兵跟在坦克后面冲锋或者防守，避免敌方火力杀伤。
2. 己方步兵优先击毙敌方手持火箭筒攻击坦克的士兵。
3. 己方步兵注意避让坦克的前进路线，不要被压到了。


步炮协同是一个小改进，在之前也稍加提到，在游戏中可以简化为：
1. 士兵在冲锋前可以自行使用火箭筒攻击冲锋区域或者敌方具有威胁的目标。当然这一步可以让坦克完成，如果有坦克。
2. 士兵在防守的时候，也要学会用适当的概率用火箭筒攻击敌方冲锋目标。当然这一步可以让坦克完成，如果有坦克。
3. 士兵要学会躲避己方炮火，己方炮火覆盖（火箭筒或者坦克或者直升机）的时候要注意避让己方士兵，除非能够做到自损1个杀敌3个。

步机协同就是让直升机和士兵更好的协同进攻，在游戏中，可以简化为：
1.  己方步兵优先击毙敌方手持火箭筒攻击坦克的士兵。
2.  己方步兵注意避让直升机打击路线，不要被自己人炸到。
3.  直升机时刻注意火力输出，支援己方士兵冲锋或者防守。

注意，步坦协同和步机协同需要至少8个己方士兵在相近范围内才能够触发，不然这就是浪费坦克和直升机。

## 步兵战术

首先，步兵战术采用典型的33制，一个squad3人，采用中国典型33制战斗部署。三个squad也可以组成一个大队，三个大队也可以组成一个连队，在往上就不设计相关战术了。在这部分，所有的战术需要参照中国人民志愿军的展示战法，并做适当的游戏简化（比如，游戏中不太可能凑的那么整齐全是333的）。

如果组成小队，一些可以加入战术：
1. 分散站位，前进、冲锋、撤退、转移的时候，需要尽可能分散但又不能离得太远，避免被一锅端。
2. 包抄和迂回。成建制的3人小队或者3squad大队，在进攻、防守、冲锋的时候，可以以36%的概率分出去三分之一，用来完成对最终目的的包抄和进攻。
3. 掩体切换：当前掩体被压制时，寻找 5-15 米外的新掩体； 交火中不再站定不动，而是向掩体方向侧移，一队 bot 交替前进——一人射击掩护，另一人移动到下一个掩体。
4. 冲锋。

一些细节和边界情况处理：
1. 可以采用ID编号的形式处理小队。如果一个小队战斗到只剩一人，那就把这个小队的ID编号抹除，这个bot采用单兵作战模式，或者就近寻找单兵bot组成新的小队，这时候给这个新小队分配新的编号。另外，bot死后重生，小队的归属不保留，重生后自行寻找小队并入。允许4-5人的小队出现，但尽量少。小队-大队和大队-连队也可以参考这样。
2. 如1.所示，采用动态的情况来管理小队、大队和连队。这对代码提出了较高的要求。可以接纳玩家进入小队，但是注意要以某种方式题型玩家。


# 随机过程

这一部分是整个修改的精髓，它彻底在创意层面让AI更加强大。

在主要目标正常的情况下（即，不跑到地图外面，攻击目标的确为地方目标），用概率限制bot做出如下举动：

1. （5%）单兵bot尝试超远距离迂回，偷取敌方载具。如果条件合适，可以偷家。
2. （2.5%）单兵bot尝试站定点位，通过弹药箱（先放在地上）+火箭筒的形式进行顶点高强度火力输出。但是注意这个的目的是服务于己方冲锋，或者大规模杀伤敌方力量。
3. （1%）单个squad全体尝试超远距离迂回，尝试偷家。
4. （0.2%）步坦协同尝试超远距离迂回，尝试偷家。
5. （0.05%）在极度劣势的情况下，整个队伍尝试所有士兵+可控制载具超远距离突围，尝试偷家。
6. （0.05%）单兵bot在开直升机的时候，如果发现对方某个点位无人值守，则跳飞机并尝试偷家。

同时，用概率限制bot做出下列举动：
1. （8%）单兵bot尝试不走寻常路，通过非既定道路到达既定目的地
2. （2%）单个squad全体不走寻常路，通过非既定道路到达既定目的地


# 重要注意事项

上述的所有内容都是对于原版游戏的加强，而不是“改版”。所以，所有AI的底层逻辑必须严格遵循原版游戏，即，尽可能杀伤敌方，尽可能冲向战场，尽可能占领重生点。
我们只能做一些如上所述的，策略上、战术上的改动。
如果我们的改动让bot出现了难以理解的异常行为，或者整个游戏出现了bug，那这是一个失败的工作。我不希望看到。
另外，你在工作的时候，注意代码逻辑的内在优雅，不要出现以“多个文件控制一个量”为代表的屎山代码。
